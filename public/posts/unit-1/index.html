Unit-1 : Overview of Markdown and Wiki:

  It was a group activity lab session, we were given 5 tasks first 4 of them were indivdual. the last task which is Task 5 it was done by group effort.
  Task 1; Understanding WIKI-
           In this task we were asked to explore how wikis function as collaborative platform where multiple users can create and edit content basically. Wikis maintain history, it's easy to track changes
           and any contributions.Through this I could understand that platforms like wiki which is a documentation sites rely on wiki based or coded systems to manage large information of data.
   Task 2: Praticing Markdown- Markdown is basically an easy light language used for formatting texts easily, I got to pratice basic syntaxs like headings, lists, bold and Italics.
          EXAMPLE; #Heading
                   **Bold Text**
                   *Italic Text*
                   - List Item
    This task showed me how markdown simplifies formatting texts.

  Task 3: Installing Zettlr-
           I installed Zettlr which is a markdown editor used for designing for writing and organizing notes efficiently. It's easy to focus on content creation. Using Zettlr it made me understand how
           markdown is used in real word for writing.

  Task 4:
  Type Setting Mathmetical Expressions- In this task we were asked to pratice writing mathmatical expressions using LaTeX format.
                                       Examples; E= mc^2

    GROUP ACTIVITY (TASK 5)
     We were assigned readings as per Group numbers we were Group 3: we had to read a books sub chapter 3 the books name was: The Information: A History, a Theory, a Flood by James Gleick.
      The sub chapter 3 name is - The Uncertainity in Our Writing, the Inconsistency in Our Letters. Honestly speaking at first the title sounded a bit abstract, but as I was going through the chapter,it
      became interesting to see how basic as writing wasn't always as structured as we assume it to be today.


      Discussion and Insights -
       What stood out to me the most was the idea that writing used to be inconsistent and uncertain, which made communication more complicated than it is now. Different spellings,lack of standard rules and variation in representation made it harder to preserve and transform information accurately. While discussing this in the group, I realized how much we take structured writing very granted nowadays. Tools like markdown, which we practiced in this lab actually solve a problem that existed for centuries the need for clarity and consistency. It was quite interesting to connect something very important to what we were
       doing pratically in the same lab. It made the learning feel more meaningful rather than just random task!


       Collaboration Experience -
                                Personally I enjoyed the group discussion because it gave me a different perspectives and I could get to know what my co group mates think about it at the same time
                                on the same chapter, some  points that I overlooked were picked up by others, whcih helped me understand the topic even more clearly.
                                at the same time putting everyone's thoughts together wasn't completely easy- we had to align our ideas and intepretations. But that process itself made the discussion
                                even more enagaging and interactive. Overall, it felt less like a formal task and more like a shared learning experience. What I found more interesting was how a simple
                                concept like markdown connects historical changes when it comes to writing, making  the lab 1 feel more pratical.
